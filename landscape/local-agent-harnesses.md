# Local Agent Harnesses

Local agent harnesses are desktop or CLI environments that run agents close to the user's machine, files, tools, and local workflows.

## Systems

- OpenClaw
- Hermes Agent
- OpenCode
- Local MCP-based agent setups
- Custom CLI agent shells

## Research Questions

- Is the harness model-agnostic?
- How does it manage local files, tools, and command execution?
- Does it provide long-term memory itself, or depend on external services?
- Can multiple agents share the same memory?
- How portable is the configuration across machines?
- What privacy guarantees come from being local-first?

## Landscape Notes

Local harnesses are often strong at tool execution and local context access. Their memory story varies widely: some rely on local files, some use protocol-based memory, and some leave long-term memory to external systems.
