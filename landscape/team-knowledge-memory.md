# Team / Enterprise Knowledge Memory

Team and enterprise knowledge memory systems organize shared company context, documents, meetings, messages, and workflows.

## Systems

- Glean
- Notion AI
- Slack AI
- Granola
- NotebookLM
- Enterprise RAG systems
- Internal knowledge graph systems

## Research Questions

- What organizational data sources are indexed?
- Is the system optimized for search, summarization, workflows, or agent memory?
- How does permissioning work?
- Can agents write new memories or only retrieve existing knowledge?
- Does memory persist at team, department, or organization scope?
- Can the memory be reused outside the product?

## Landscape Notes

Enterprise knowledge systems are strong at shared context and permissioned search. The open question is how much of that context becomes agent-native memory rather than a human-facing knowledge base.
