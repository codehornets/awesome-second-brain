# Coding Agents & Developer Harnesses

Coding agents and developer harnesses use repo context, rules, tool access, session history, and local configuration to make software engineering agents more effective.

## Systems

- [Claude Code](../systems/claude-code.md)
- Codex
- Cursor
- Windsurf
- GitHub Copilot
- Gemini CLI
- OpenCode

## Research Questions

- How does the agent read codebase context?
- Does it support rules, instructions, skills, or project memory files?
- Does memory persist across sessions?
- Can it connect to external memory through MCP or other tool protocols?
- Is memory local, cloud-hosted, or platform-managed?
- Can teams share context, or is it user-specific?

## Landscape Notes

Developer memory is often a mix of explicit files, hidden product state, chat history, and tool connectors. This makes it powerful but hard to compare without a standard template.
